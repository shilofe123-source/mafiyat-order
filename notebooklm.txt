# הסבר: mafiyat-order — מערכת הזמנות מגשי אירוח למאפיית השומרון

## מה זה עושה

מערכת הזמנות מקוונת עבור "בית קפה מאפיית השומרון" (קרני שומרון, כשרות מהדרין רבנות קרני שומרון).
המערכת מאפשרת ללקוחות לבחור מגשי אירוח, למלא פרטים, לקבל ייעוץ מבוט AI, ולשלוח את ההזמנה כ-PDF בוואטסאפ — ובמקביל לשמור הכל בגיליון Google Sheets.

המערכת מיועדת לשני שימושים עיקריים:
- לקוחות בונים הזמנה באופן עצמאי דרך ממשק אינטרנטי
- בעלי העסק (אורי ובת חן) משתמשים ב"מצב עורך" כדי להגדיר הנחות ולאשר הזמנות

---

## מבנה

### index.html — האפליקציה המלאה (קובץ יחיד, ~993 שורות)
קובץ HTML אחד שמכיל CSS, לוגיקה, ותצוגה. לא תלוי בשרת — רץ ישירות מ-GitHub Pages.

חלקים עיקריים בקובץ:

- **CSS (שורות 15-124):** עיצוב כחול-שמנת (gradient של כחול כהה ושמנת), עיצוב הצ'אט AI, תמיכה ב-responsive (מובייל/דסקטופ), ותמיכה בהדפסה/PDF.

- **קונפיגורציה וקבועים (שורות 134-192):** מכיל PIN למצב עורך (2001), מספרי טלפון של הבעלים (אורי ובת חן), כתובת Google Sheets, לוגו מוטמע כ-base64, ומערך PRODUCTS הראשי עם 30 מוצרים ב-6 קטגוריות.

- **קומפוננטות React (שורות 194-322):**
  - Toast — הודעות צפות (למשל "ההזמנה נשמרה")
  - GearButton — כפתור גלגל שיניים לכניסה למצב עורך
  - PinModal — חלון הזנת PIN
  - ApprovalModal — חלון אישור הזמנה עם סיכום מחירים ואפשרות הנחה

- **פונקציות עזר (שורות 327-362):**
  - normalizePhone — ממיר מספר טלפון ישראלי לפורמט בינלאומי (972...)
  - generatePdfAndShare — יוצר PDF מאלמנט HTML באמצעות html2pdf.js, מנסה לשתף דרך Web Share API, ואם לא — מוריד את הקובץ ופותח וואטסאפ

- **App — הקומפוננטה הראשית (שורות 364-740):** מכילה שני מצבי תצוגה:
  - build — בניית ההזמנה: טופס פרטי לקוח, תפריט מוצרים עם חיפוש וסינון לפי קטגוריה, סל הזמנה עם ניהול כמויות
  - preview — תצוגה מקדימה של הצעת המחיר בפורמט טבלאי מעוצב, עם אפשרות לאשר, לשלוח PDF בוואטסאפ לאורי/בת חן/הלקוח

- **פאנל צ'אט AI (שורות 743-993):** ממשק צ'אט נפרד בצד ימין (בדסקטופ) או בתחתית (במובייל). הצ'אט כולל מערך PRODUCTS שני ומערך RULES שמגדיר כללי ייעוץ (לפי עונה, סוג אירוע, גיל האורחים). הצ'אט פונה ל-API של Base44, ובמקרה של כשל — נופל ל-fallback מקומי עם תשובות מוכנות מראש.

### Code.gs — Google Apps Script (104 שורות)
סקריפט שרץ בצד Google ומשמש כ-backend:
- doPost — מקבל הזמנות מהטופס ושומר אותן בגיליון "הזמנות" (12 עמודות: חותמת זמן, תאריך, שם, טלפון, אירוע, אורחים, פריטים, מחירים, הנחה, הערות)
- callGeminiVision — מקבל תמונה ב-base64 ושולח אותה ל-Gemini 1.5 Flash לקבלת תיאור בעברית
- doGet — endpoint בדיקת חיות שמחזיר סטטוס "ok"

### functions/bakeryChat.ts — שרת צ'אט Deno (67 שורות)
שרת Deno שרץ על פלטפורמת Base44. מקבל הודעת לקוח + היסטוריית שיחה + system prompt, ומעביר הכל ל-API של Anthropic (מודל claude-haiku-4-5). כולל CORS headers לתמיכה בקריאות cross-origin מהדפדפן.

### mafiyat_hashomron_system.md — מסמך תכנון המערכת
מסמך ראייה שמתאר את המערכת המלאה כפי שתוכננה: בוט AI שמנהל שיחות עם לקוחות, ניהול מלאי ב-Google Sheets, אוטומציות (תזכורות לבת חן, שאלוני משוב), ותכנית לחיבור WhatsApp Business API.

### README.md — תיעוד
הנחיות התקנה, תיאור אינטגרציות, והוראות לעדכון מוצרים ומחירים.

## זרימת הנתונים

### זרימת הזמנה רגילה:
1. הלקוח נכנס לאתר (GitHub Pages) ורואה את טופס ההזמנה
2. ממלא פרטים אישיים (שם, טלפון, תאריך, סוג אירוע, מספר אורחים)
3. בוחר מוצרים מתוך התפריט — אפשר לסנן לפי קטגוריה או לחפש
4. לוחץ "ביצוע ההזמנה" — אם CONFIG.CLIENT_SELF_APPROVE=true נפתח חלון אישור ישירות, אחרת מוצגת תצוגה מקדימה
5. בתצוגה מקדימה — לוחץ "אישור"
6. במצב עורך — אפשר להגדיר הנחה לפני האישור
7. לאחר אישור — הנתונים נשלחים ב-POST ל-Google Apps Script שכותב שורה חדשה בגיליון
8. נפתח חלון "שליחת הצעת מחיר" — אפשר לשלוח PDF בוואטסאפ לאורי, לבת חן, או ללקוח עצמו
9. ה-PDF נוצר מתוך תצוגת ה-preview באמצעות html2pdf.js, ונשלח דרך Web Share API או כהורדה + קישור וואטסאפ

### זרימת צ'אט AI:
1. הלקוח כותב הודעה בפאנל הצ'אט
2. ההודעה נשלחת ל-Base44 bakeryChat (שמעביר ל-Anthropic Claude Haiku)
3. אם ה-API נכשל — נופל ל-fallback מקומי שמחזיר תשובות לפי מילות מפתח
4. אם הלקוח סימן "אפשר לסוכן למלא את ההזמנה בשבילי" — ה-AI יכול להחזיר JSON עם נתוני מילוי טופס
5. הצ'אט שומר היסטוריית 10 הודעות אחרונות כהקשר

## מושגים מרכזיים

### HTM (Hyperscript Tagged Markup)
תחליף ל-JSX שלא דורש צעד בנייה (build step). במקום לכתוב JSX ולהריץ Babel, משתמשים ב-tagged template literals. לכן אין צורך ב-bundler — הכל רץ ישירות בדפדפן.

### מצב עורך (Editor Mode)
מצב מוגן ב-PIN שמאפשר לבעלי העסק להגדיר הנחות להזמנות, להעלות תמונות (שעוברות ניתוח עם Gemini Vision), ולשלוט בתהליך האישור.

### CLIENT_SELF_APPROVE
דגל שקובע אם הלקוח יכול לאשר הזמנה ישירות (true) או שחייב לעבור קודם תצוגה מקדימה (false). כרגע מוגדר false.

### Fallback מקומי בצ'אט
מנגנון שמבטיח שהצ'אט תמיד עובד גם אם ה-API של Base44 לא זמין. מכיל תשובות מוכנות מראש לנושאים נפוצים: ברכה, כשרות, ילדים, חורף, מחירים, ארוחת בוקר, וחישוב כמויות.

### PDF + WhatsApp Flow
שילוב של html2pdf.js (יוצר PDF מ-HTML) עם Web Share API (שיתוף מקורי במכשירי מובייל) ו-WhatsApp deep links (wa.me). אם Web Share לא נתמך — הקובץ מורד למכשיר והלקוח מועבר לוואטסאפ עם הנחיה לצרף את הקובץ.

### שני מערכי PRODUCTS
מערך אחד (שורה ~149) משמש את טופס ההזמנה עם שדות id/name/price/qty_desc/cat. מערך שני (שורה ~771) משמש את הצ'אט AI עם שדות נוספים כמו tags ו-category. בעת עדכון מוצרים — חובה לעדכן בשניהם.

## הערות נוספות

- **הכל בקובץ אחד:** index.html מכיל CSS + JavaScript + HTML ללא תלות בנייה. זה מאפשר עריכה ישירה ב-GitHub ופריסה מיידית דרך GitHub Pages.

- **לוגו מוטמע:** הלוגו של המאפייה מוטמע כ-base64 data URI ישירות בקוד, כך שאין צורך בקובץ תמונה נפרד.

- **טיפול ב-RTL ו-PDF:** יש בעיה ידועה שב-PDF הנוצר עם html2pdf.js, מילים בעברית מאבדות רווחים. התיקון כולל word-spacing, letter-spacing, text-rendering: geometricPrecision, letterRendering: true ב-html2canvas, ו-direction: rtl + unicode-bidi: embed על תאי הטבלה.

- **מספרי טלפון קבועים בקוד:** מספרי הטלפון של אורי (0524767233) ובת חן (0542031448) קבועים בקוד כ-PHONES object.

- **Gemini API Key חשוף:** מפתח ה-API של Gemini נמצא בטקסט גלוי ב-Code.gs. זה מוגן חלקית כי Code.gs רץ בצד השרת של Google, אבל הקובץ עצמו גלוי ב-GitHub.

- **מצב נוכחי vs תוכנית:** מסמך התכנון (mafiyat_hashomron_system.md) מתאר חזון רחב יותר שכולל ניהול מלאי, תזכורות אוטומטיות, שאלוני משוב, וחיבור WhatsApp Business API. כרגע מומש רק טופס ההזמנה + צ'אט AI + שמירה בגיליון.

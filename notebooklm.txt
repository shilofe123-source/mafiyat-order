# הסבר: מאפיית השומרון — ממשק הלקוח וממשק העורך

## מה זה עושה

מערכת הזמנות מגשי אירוח למאפיית השומרון (קרני שומרון, כשרות מהדרין רבנות קרני שומרון).
המערכת מאפשרת ללקוחות לבחור מוצרים, למלא פרטים, ולשלוח הזמנה שנשמרת בגיליון Google Sheets.
כוללת צ'אט AI מובנה שיכול לייעץ ולמלא את הטופס אוטומטית, ומצב עורך מוגן בסיסמה לניהול הזמנות.

האפליקציה בנויה כקובץ HTML יחיד ללא שרת — React 18 דרך CDN עם תחביר HTM (ללא build step).

---

## מבנה

### קבצים

- index.html — האפליקציה המלאה: ממשק לקוח, ממשק עורך, צ'אט AI, וכל הלוגיקה. כ-816 שורות.
- Code.gs — Google Apps Script: שמירת הזמנות בגיליון, חילוץ הזמנות מתמונות ו-PDF באמצעות Gemini 2.5 Flash.
- functions/bakeryChat.ts — Deno endpoint לצ'אט AI, רץ על שרת Base44, מתקשר עם Claude Sonnet 4.6.
- images/ — 27 תמונות מוצרים דחוסות המוצגות ברקע המשתנה.

## ממשק הלקוח

### מה הלקוח רואה
עמוד יחיד עם רקע משתנה (slideshow של תמונות מוצרים כל 8 שניות) ושכבת שקיפות לקריאות. העיצוב רספונסיבי — מותאם למובייל ולדסקטופ.

### תהליך הזמנה
1. הלקוח בוחר מוצרים מתוך רשימה מחולקת לקטגוריות: מגשי אירוח, מאפים חמים, מנות חמות, פלטות, סלטים, פירות וקינוחים.
2. לכל מוצר יש שם, מחיר, ותיאור כמות (למשל "12 יחידות").
3. הלקוח ממלא פרטים: שם, טלפון, סוג אירוע, מספר אורחים, תאריך, והערות.
4. לאחר אישור — ההזמנה נשמרת בגיליון Google Sheets, ונוצר קובץ PDF עם הצעת מחיר.
5. הלקוח יכול לשלוח את ה-PDF בוואטסאפ ישירות מהאפליקציה.

### צ'אט AI
בצד ימין של המסך (או כפאנל נפתח במובייל) יש צ'אט עם סוכן AI. הסוכן:
- ממליץ על מגשים לפי סוג אירוע, עונה, וכמות אורחים
- מכיר את כל המוצרים, המחירים, והכשרות
- יכול למלא את טופס ההזמנה אוטומטית (אם הלקוח מסמן "אפשר לסוכן למלא")
- מקבל תמונות ו-PDF דרך כפתור צירוף קבצים (סיכת נייר), עד 4MB

כשהסוכן ממלא טופס, הוא מחזיר JSON מובנה שמפורסר על ידי פונקציית bracket-counting (לא רגקס) ומועבר לטופס. הלקוח לעולם לא רואה JSON גולמי — אם הפרסור נכשל, ה-JSON מוסר מהתצוגה.

### Preview
לפני שליחת ההזמנה, הלקוח רואה תצוגה מקדימה של ה-PDF עם כל הפרטים, המוצרים, והסכום.

## ממשק העורך

### גישה
ממשק העורך מוגן בקוד PIN (S199). לחיצה על אייקון הגדרות בפינת המסך פותחת חלון הזנת קוד.

### יכולות העורך

1. **הנחות** — העורך יכול להגדיר אחוז הנחה שמחושב אוטומטית על הסכום הסופי. שדה ההנחה מוצג רק במצב עורך.

2. **העלאת קבצים לחילוץ הזמנה** — העורך יכול להעלות תמונה או PDF של הזמנה קיימת (למשל צילום מסך מוואטסאפ). הקובץ נשלח ל-Google Apps Script שמעבד אותו עם Gemini 2.5 Flash ומחלץ את הפריטים והפרטים אוטומטית לטופס. מגבלת גודל: 4MB.

3. **עדכון הזמנה קיימת** — כשהעורך שומר הזמנה, המערכת בודקת אם קיימת הזמנה עם אותו מזהה ייחודי (עמודה M בגיליון). אם כן — ההזמנה מתעדכנת במקום להיכתב כשורה חדשה. זה מאפשר לערוך הזמנות שכבר נשלחו.

4. **מידע debug** — אם חילוץ הזמנה מקובץ נכשל, העורך רואה הודעת שגיאה מפורטת (סוג השגיאה, תגובת ה-API) במקום הודעה גנרית.

## זרימת הנתונים

### הזמנה רגילה
לקוח בוחר מוצרים וממלא פרטים, לוחץ "שלח". JavaScript שולח POST ל-Google Apps Script. Script שומר שורה בגיליון "הזמנות" (13 עמודות: חותמת זמן, תאריך, שם, טלפון, אירוע, אורחים, פריטים, סכום, הנחה, סכום סופי, הערות, אישור, מזהה). Script מעדכן גיליון "סיכום" (שם לקוח + סכום סופי). הפרונטנד מייצר PDF עם html2pdf.js. הלקוח שולח בוואטסאפ.

### צ'אט AI
לקוח כותב הודעה (ואופציונלית מצרף תמונה/PDF). JavaScript שולח POST ל-Base44 bakeryChat endpoint. ה-Endpoint שולח ל-Claude API עם system prompt שמכיל את רשימת המוצרים. Claude מחזיר תשובה — טקסט, או JSON עם fill אם מילוי מאופשר. הפרונטנד מפרסר ומציג או ממלא.

אם ה-API של Base44 לא זמין — יש fallback מקומי שמתאים תשובות מוכנות מראש לפי מילות מפתח בהודעת הלקוח.

### חילוץ מקובץ (מצב עורך)
עורך מעלה קובץ. JavaScript ממיר ל-base64, שולח POST ל-Apps Script. Script שולח ל-Gemini 2.5 Flash API עם הנחיות לחלץ פריטים. Gemini מחזיר JSON. Script מפרסר ומחזיר פריטי הזמנה. הפרונטנד ממלא את הטופס.

## מושגים מרכזיים

### גשר React ← Vanilla JS
הצ'אט כתוב ב-JavaScript רגיל (vanilla), בעוד הטופס הוא קומפוננטת React. כדי שהצ'אט יוכל למלא את הטופס, React חושף פונקציות גלובליות על window:
- הוספת מוצרים לעגלה (מיזוג עם קיימים)
- הסרת מוצרים לפי מזהה
- החלפת כל העגלה
- מילוי פרטי לקוח (שם, טלפון, תאריך וכו')

הפונקציות משתמשות ב-functional updates של React (prev => ...) כדי להימנע מבעיות closure.

### מערך מוצרים יחיד
יש מערך מוצרים ראשי אחד בלבד. מערך המוצרים של הצ'אט נגזר ממנו באמצעות map עם טבלאות בדיקה (lookup) לתגיות וכמויות — אין כפילות, אין בעיות סנכרון.

### Bracket-counting parser
במקום רגקס שביר, הקוד סופר סוגריים מסולסלים כדי לחלץ JSON מתשובת ה-AI. זה עובד גם עם JSON מעוצב (pretty-printed), קינון עמוק, ותווים מיוחדים בתוך מחרוזות.

### stripJson
פונקציה שמסירה בלוקי JSON מתשובות ה-AI לפני הצגה ללקוח — מבטיחה שהלקוח לעולם לא רואה JSON גולמי.

### מזהה הזמנה ייחודי
כל הזמנה מקבלת מזהה ייחודי (orderId) שנשמר בעמודה M בגיליון. במצב עורך, שמירה חוזרת עם אותו מזהה מעדכנת את השורה הקיימת במקום ליצור שורה חדשה.

### HTM (Hyperscript Tagged Markup)
תחליף ל-JSX שלא דורש צעד בנייה. במקום לכתוב JSX ולהריץ Babel, משתמשים ב-tagged template literals. הכל רץ ישירות בדפדפן.

### Fallback מקומי
מנגנון שמבטיח שהצ'אט תמיד עובד גם אם ה-API של Base44 לא זמין. מכיל תשובות מוכנות מראש לנושאים נפוצים.

## תלויות חיצוניות

- React 18 + ReactDOM (CDN) — ספריית UI
- HTM (CDN) — תחליף ל-JSX ללא build step
- html2pdf.js 0.10.1 (CDN) — יצירת PDF מ-HTML
- Google Apps Script — backend לשמירת הזמנות וחילוץ מקבצים
- Google Sheets — מסד נתונים להזמנות (גיליון "הזמנות" + גיליון "סיכום")
- Gemini 2.5 Flash API — חילוץ הזמנות מתמונות ו-PDF (נקרא מ-Apps Script, מפתח ב-Script Properties)
- Base44 Platform — hosting ל-bakeryChat endpoint
- Claude Sonnet 4.6 API — מנוע ה-AI של הצ'אט (נקרא מ-bakeryChat endpoint)
- WhatsApp (wa.me) — שליחת PDF ללקוח ולבעלים
- GitHub Pages — hosting לאתר

## הערות נוספות

- הכל בקובץ אחד: index.html מכיל CSS + JavaScript + HTML ללא תלות בנייה. זה מאפשר עריכה ישירה ב-GitHub ופריסה מיידית דרך GitHub Pages.

- הלוגו של המאפייה מוטמע כ-base64 data URI ישירות בקוד, כך שאין צורך בקובץ תמונה נפרד.

- טיפול ב-RTL ו-PDF: יש בעיה ידועה שב-PDF הנוצר עם html2pdf.js, מילים בעברית מאבדות רווחים. התיקון כולל word-spacing, letter-spacing, text-rendering: geometricPrecision, letterRendering: true, ו-direction: rtl + unicode-bidi.

- מפתח ה-API של Gemini מאוחסן ב-Script Properties של Google Apps Script (לא בקוד), אחרי תיקון דליפת אבטחה.

- ה-system prompt של הצ'אט כולל את כל מזהי המוצרים כדי שהסוכן יוכל לייצר JSON תקין עם id נכון.

- שדה התאריך הוא שדה חובה. הסוכן יכול למלא אותו, ויש המרה אוטומטית מפורמט DD/MM/YYYY לפורמט YYYY-MM-DD.

- הרקע המשתנה מציג תמונות מוצרים אמיתיות מתיקיית images/ — 27 תמונות שמתחלפות כל 8 שניות עם אנימציית fade.

- תמיכה בצירוף קבצים לצ'אט: תמונות ו-PDF עד 4MB, נשלחים כ-base64 ל-Claude API. PDF משתמש ב-type: "document" (לא "image").
